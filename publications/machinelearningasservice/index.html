Running Machine Learning as Service

Link to full Talk

This talk looked at the nuances of deployment of Machine Learning models in production. In current era, when CI & CD are supported for almost all type of applications developed and with integration with services like GCP, AWS; one can deploy scalable solution at instant.

However, the idiosyncratic complexity for deployment of Machine learning model in robust system is still a mystry and many firms have invested for add-on frameworks for developing a comparable solution to push it to production environemnt, but many still face the challanges like scalability, latency etc.

We propose a python heavy environment for deploying the ML models in service layer with flexibility and atmost latency.

Why Machine Learning as Service?

Limited Scale

  • Model created with Python*
  • Great for prototypes, but same thing works horribly in Prod
  • Not-so-great with existing ecosystems

What’s more

  • Built for team-specific use cases
  • No Centralized Algorithm Store, Same problems get solved again and again

Challenges

Non-reproducible Models

  • Extremely Hard to identify anomaly behavior, esp. DNN
  • Model Versioning
  • Tracking Model Quality and OPX

Motivation

  • Michelangelo, Uber Michelangelo
  • MIT Model DB, ModelDB

Proposal

Code it, Stream it.

Struct the model and stream to over the service layer and API will be up upon completion
